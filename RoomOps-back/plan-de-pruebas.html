Plan de Pruebas — RoomOps Backend

Proyecto: RoomOps  |  Módulo: Backend (Spring Boot 3.3.5 - Java 17)  |  Versión: 1.0  |  Fecha: mayo 2026  |  Tipo: Pruebas Automatizadas

1. Objetivos del Plan

El presente plan de pruebas tiene como finalidad verificar que las funcionalidades implementadas en el backend de RoomOps se comporten correctamente según los requerimientos definidos, cubriendo los controladores REST, servicios de negocio, autenticación JWT, autorización basada en roles (RBAC) y persistencia de datos en PostgreSQL.

Las pruebas son de carácter automatizado, ejecutadas mediante JUnit 5, Mockito y Spring Test Framework, con cobertura medida mediante JaCoCo.

10. Mejores Prácticas

10.1 Patrón AAA (Arrange-Act-Assert)

Arrange: Preparar datos y contexto  |  Act: Ejecutar acción  |  Assert: Verificar resultados

10.2 Independencia de Pruebas

  • Cada prueba debe ser independiente de otras
  • Usar setUp() y tearDown() para limpiar estado
  • No debe haber dependencias de orden de ejecución

10.3 Nombres Descriptivos

✓ testGetUsersReturnsListWhenDatabaseNotEmpty()

✗ test1()

10.4 Evitar Pruebas Frágiles

  • No usar esperas (Thread.sleep) con valores fijos
  • Evitar dependencias externas en unitarias
  • Usar fixtures y builders para datos complejos

11. Conclusiones

Este plan de pruebas establece un marco sólido para garantizar la calidad del backend de RoomOps. La combinación de pruebas unitarias, de integración y de seguridad proporciona cobertura integral del sistema.

Se espera que la implementación sistemática de este plan resulte en:

  • Mínimo 80% de cobertura de código en componentes críticos
  • Sistema seguro con autenticación y autorización validadas
  • Confianza en la confiabilidad del sistema
  • Facilitar mantenimiento y refactorización futura
  • Permitir despliegue seguro en producción

La ejecución regular de estas pruebas debe ser parte integral del ciclo de desarrollo, siendo ejecutadas en cada commit y como parte del pipeline CI/CD.
