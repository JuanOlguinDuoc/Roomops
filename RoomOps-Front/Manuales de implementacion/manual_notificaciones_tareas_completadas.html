Manual de implementacion: notificaciones de tareas completadas

Objetivo

Implementar en el dashboard admin:

  1. Un punto rojo en el boton de notificaciones cuando una tarea pase a completada.
  2. Un popup con SweetAlert2 al presionar el boton, mostrando las tareas completadas.
  3. Marcar esas notificaciones como leidas despues de verlas.

Resumen funcional

Explicacion no tecnica

Piensalo como una bandeja de alertas:

  1. El sistema revisa si una tarea cambio de estado a completada.
  2. Si detecta ese cambio, crea una notificacion nueva.
  3. Si existen notificaciones no leidas, muestra el punto rojo en la campana.
  4. Al tocar la campana, se abre una ventana con el listado de tareas completadas.
  5. Al cerrar esa ventana, las notificaciones se marcan como leidas y desaparece el punto rojo.

Explicacion tecnica

Se implementa con tres piezas:

  1. Estado React para manejar notificaciones en memoria.
  2. localStorage para persistir notificaciones y snapshot de estado por tarea entre recargas.
  3. Deteccion de transicion de estado (de no done a done) para crear notificaciones solo cuando hay cambio real.

Paso 2: crear claves de localStorage

Que agregar

Define constantes cerca de tus constantes globales:

const COMPLETED_NOTIFICATIONS_KEY = 'roomops-completed-notifications'
const TASK_STATUS_SNAPSHOT_KEY = 'roomops-task-status-snapshot'

Por que

  • No tecnico: para que los avisos no se pierdan al recargar la pagina.
  • Tecnico: se guardan notificaciones y el ultimo estado conocido por taskId para detectar transiciones.

Paso 3: crear estado para notificaciones

Que agregar

Dentro del componente Home (dashboard admin), agrega estado:

const [notifications, setNotifications] = useState([])

Y un useEffect de carga inicial:

useEffect(() => {
  try {
    const raw = window.localStorage.getItem(COMPLETED_NOTIFICATIONS_KEY)
    const parsed = raw ? JSON.parse(raw) : []
    setNotifications(Array.isArray(parsed) ? parsed : [])
  } catch {
    setNotifications([])
  }
}, [])

Por que

  • No tecnico: esto es la bandeja interna de avisos.
  • Tecnico: el estado React permite render condicional del punto rojo y refresco automatico de la UI.

Paso 4: detectar cuando una tarea pasa a completada

Que agregar

Crea un useEffect que observe tareas visibles y statusById.

Logica recomendada:

  1. Construir mapa actual: taskId -> statusKey con getTaskStatusKey.
  2. Leer snapshot previo desde localStorage.
  3. Comparar cada tarea:
    • si previo != done y actual == done => crear notificacion.
  4. Guardar snapshot nuevo.
  5. Guardar notificaciones nuevas en estado y localStorage.

Ejemplo base:

useEffect(() => {
  if (!Array.isArray(tasks) || tasks.length === 0) return

  const currentSnapshot = {}
  tasks.forEach((task) => {
    if (task?.id == null) return
    currentSnapshot[String(task.id)] = getTaskStatusKey(task, statusById)
  })

  const rawPrev = window.localStorage.getItem(TASK_STATUS_SNAPSHOT_KEY)
  const prevSnapshot = rawPrev ? JSON.parse(rawPrev) : null

  // Primera carga: solo guardar snapshot, sin generar notificaciones historicas
  if (!prevSnapshot || typeof prevSnapshot !== 'object') {
    window.localStorage.setItem(TASK_STATUS_SNAPSHOT_KEY, JSON.stringify(currentSnapshot))
    return
  }

  const justCompleted = []

  tasks.forEach((task) => {
    if (task?.id == null) return

    const id = String(task.id)
    const prevStatus = prevSnapshot[id]
    const nextStatus = currentSnapshot[id]

    if (prevStatus !== 'done' && nextStatus === 'done') {
      justCompleted.push({
        id: `${id}-${Date.now()}`,
        taskId: task.id,
        title: task.titulo || 'Tarea sin titulo',
        completedAt: new Date().toISOString(),
        read: false
      })
    }
  })

  if (justCompleted.length > 0) {
    setNotifications((prev) => {
      const next = [...justCompleted, ...prev].slice(0, 50)
      window.localStorage.setItem(COMPLETED_NOTIFICATIONS_KEY, JSON.stringify(next))
      return next
    })
  }

  window.localStorage.setItem(TASK_STATUS_SNAPSHOT_KEY, JSON.stringify(currentSnapshot))
}, [tasks, statusById])

Por que

  • No tecnico: compara un antes y un despues para saber que tarea se completo recien.
  • Tecnico: evita falsos positivos; una tarea ya completada no debe volver a notificar en cada render.

Paso 6: crear handler del popup SweetAlert2

Que agregar

Crea una funcion handleOpenNotifications:

const handleOpenNotifications = async () => {
  if (!notifications.length) {
    await Swal.fire({
      icon: 'info',
      title: 'Notificaciones',
      text: 'No hay tareas completadas para mostrar.'
    })
    return
  }

  const itemsHtml = notifications
    .slice(0, 20)
    .map((n) => {
      const dateText = new Date(n.completedAt).toLocaleString('es-CL')
      return `<li style="margin-bottom:8px;"><strong>${n.title}</strong><br/><small>${dateText}</small></li>`
    })
    .join('')

  await Swal.fire({
    icon: 'success',
    title: 'Tareas completadas',
    html: `<ul style="text-align:left;padding-left:18px;max-height:280px;overflow:auto;">${itemsHtml}</ul>`,
    confirmButtonText: 'Entendido'
  })

  setNotifications((prev) => {
    const next = prev.map((item) => ({ ...item, read: true }))
    window.localStorage.setItem(COMPLETED_NOTIFICATIONS_KEY, JSON.stringify(next))
    return next
  })
}

Por que

  • No tecnico: al tocar la campana ves el resumen de tareas completadas.
  • Tecnico: el popup consume notificaciones y sincroniza estado read en memoria y persistencia.

Paso 7: conectar boton de campana

Que modificar

En el boton actual de Home.jsx:

Paso 9: pruebas recomendadas

  1. Carga Home con tareas no completadas.
  2. Completa una tarea desde tu flujo actual.
  3. Verifica que aparece el punto rojo en la campana.
  4. Presiona campana y valida que aparece la tarea en el popup.
  5. Cierra popup y verifica que desaparece el punto rojo.
  6. Recarga la pagina y confirma que el estado leido se mantiene.
  7. Completa otra tarea y confirma que vuelve a aparecer el punto rojo.

Errores comunes y como evitarlos

  1. Error: se generan notificaciones viejas al cargar por primera vez.
  • Causa: no distinguir primera carga.
  • Solucion: si no hay snapshot previo, solo guardar snapshot y salir.
  1. Error: el punto rojo nunca desaparece.
  • Causa: no marcar read o no persistir despues del popup.
  • Solucion: actualizar notifications y localStorage al cerrar Swal.
  1. Error: se duplica la misma notificacion.
  • Causa: detectar done sin comparar estado previo.
  • Solucion: validar transicion exacta previo != done y actual == done.

Resultado esperado

Cuando implementes estos pasos:

  1. La campana se comporta como centro de notificaciones real.
  2. El usuario recibe feedback visual inmediato (punto rojo).
  3. Puede consultar completadas en popup SweetAlert2.
  4. El sistema mantiene historial y estado de lectura incluso con recargas.

Fix 3: siembra inicial de notificaciones (bootstrap)

Lineas modificadas en Home.jsx:

  1. 784
  2. 787
  3. 797
  4. 807

Explicacion no tecnica:

  1. Si entrabas al dashboard y ya habia tareas completadas, la campana aparecia vacia.
  2. Ahora, en la primera carga, el sistema "siembra" esas tareas completadas como notificaciones iniciales.

Explicacion tecnica:

  1. Se detecta completedTasks usando getTaskStatusKey(task, statusById) === 'done'.
  2. Si no existe snapshot previo y no hay historial cargado, se genera arreglo seeded y se persiste.
  3. El historial queda limitado a 50 para evitar crecimiento indefinido.

Codigo aplicado:

const completedTasks = tasks.filter((task) => getTaskStatusKey(task, statusById) === 'done')

if (!prevSnapshot || typeof prevSnapshot !== 'object') {
  setNotifications((prev) => {
    if (Array.isArray(prev) && prev.length > 0) return prev
    if (completedTasks.length === 0) return prev

    const seeded = completedTasks
      .slice(0, 50)
      .map((task, index) => ({
        id: `seed-${task.id}-${index}`,
        taskId: task.id,
        title: task.titulo || `Tarea ${task.id}`,
        completedAt: new Date().toISOString(),
        read: false
      }))

    window.localStorage.setItem(COMPLETED_NOTIFICATIONS_KEY, JSON.stringify(seeded))
    return seeded
  })

  window.localStorage.setItem(TASK_STATUS_SNAPSHOT_KEY, JSON.stringify(currentSnapshot))
  return
}
